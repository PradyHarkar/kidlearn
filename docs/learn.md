# Learn.md

## Purpose
This document defines the learning experience for KidLearn.

The learn experience should feel interactive, guided, age-appropriate, and rewarding.
It should never feel like a dull quiz wall.

## Product Goal
Turn learning into a guided adventure where:
- the child understands the concept
- the child feels progress
- the child gets immediate feedback
- the child can resume later
- the parent can see clear improvement

## Core Experience Principles
1. Every screen should have a clear purpose.
2. One action should feel primary on each screen.
3. Use visuals before text when the child is young.
4. Keep the tone encouraging.
5. Show growth, not just correctness.
6. Make the theme part of the experience.
7. Keep the child moving forward without confusion.

## Learning Modes
The learn flow should support:
- tap-card multiple choice
- counting scenes
- connect-the-dots
- matching
- story scenes
- writing steps
- drag-and-sort in the future

The right mode should be chosen based on:
- subject
- year
- topic
- child theme
- learning objective

## Theme Rules
The child's selected theme should carry through:
- dashboard
- learning screen
- question card
- buttons
- progress strip
- results screen
- history screen

If the theme is light, the UI should use stronger dark text.
If the theme is dark, the UI should use light text and stronger contrast.

## Question Screen Rules
- The question area should be the visual focus of the page.
- The question should be easy to read.
- The answer area should be obvious and touch-friendly.
- The submit button should never disappear into the background.
- The child should always know what to do next.

## History and Progress
The learn experience should show:
- what the child learned
- when they learned it
- how long each question took
- which topics they are strong in
- which topics need more practice

## Reward Rules
The child should earn rewards for:
- completing questions
- improving on a topic
- finishing a session
- revising work
- returning to learn again

Rewards should use the existing points system unless a feature specifically needs its own counter.

## Resume Rules
- The child should be able to continue where they left off.
- Resume state should be tied to the account and child, not only the device.
- Returning to a session should restore the current progress reliably.

## Writing Support
Writing is its own guided experience inside learn.

It should:
- avoid blank-page fear
- use one step per screen
- provide encouraging feedback
- show revision and improvement
- keep narrative and persuasive modes separate

## Age-Appropriate Behavior
- Younger kids need larger visuals and simpler language
- Middle years need guided reasoning
- Older kids need stronger challenge and clearer academic depth

## What Learn Must Never Do
- Do not hide the main action button
- Do not make the theme harm readability
- Do not drop the child into a blank page
- Do not use the same feel for every year
- Do not reduce everything to plain multiple choice
- Do not make questions feel random or disconnected

## Success Criteria
The learn experience is good when:
- the child understands the task quickly
- the question feels meaningful
- the child can answer confidently
- the child wants to continue
- the parent can see progress
- the UI feels alive and fun
